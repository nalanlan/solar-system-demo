# 太阳系科普网站协作提示词

这份文档用于团队协作参考，分为两类提示词：

- Product Design：用于生成产品方案、信息架构、视觉规范和交互设计。
- Codex：用于基于方案开发可运行前端 Demo、修复问题、打包和上传 GitHub。

项目仓库：`solar-system-demo`

## 一、Product Design 提示词

```text
你现在作为一名资深产品设计师、科普网站信息架构师、前端交互体验设计师和科幻视觉设计专家，帮我设计一个“太阳系科普网站”的完整产品方案。

项目目标：
设计一个面向青少年、科普爱好者、学校展示和公众科普场景的网站，用炫酷、沉浸式、科幻感的方式介绍太阳系，让用户能够直观了解太阳、八大行星、矮行星、小行星带、彗星、卫星、轨道运行、行星对比和宇宙探索知识。

整体风格：
1. 参考“深空探索控制台 + 全息星图 + 星际航行驾驶舱”。
2. 视觉关键词：深空黑、星云蓝、科技青、星云紫、太阳金、玻璃拟态、HUD、轨道线、粒子、辉光。
3. 页面要有高级感和宣传级视觉效果，不要做成普通百科页面。
4. 内容表达要通俗、简洁、严谨，适合非专业用户阅读。

核心页面：
1. 首页：全屏 3D Hero，标题“探索太阳系的奥秘”，副标题“跟随行星的轨道，穿越太阳、地球、火星与遥远的海王星，开启一场动态星际科普之旅。”，按钮“开始 3D 探索”“查看行星图鉴”。
2. 太阳系总览页：3D 太阳系地图，支持拖拽旋转、滚轮缩放、点击行星聚焦、速度控制、轨道/标签显示切换、HUD 数据面板。
3. 行星图鉴页：八大行星玻璃拟态卡片，包含行星图、名称、一句话特点、标签、查看详情按钮。
4. 行星详情页：每颗行星独立 3D 主视觉，包含核心数据、科普介绍、地球对比、探索任务和下一颗行星入口。

必须强化的 3D 体验：
1. 首页和总览页必须是真 3D WebGL 场景，不是静态背景图。
2. 太阳持续发光并缓慢自转。
3. 八大行星必须沿轨道公转，同时围绕自身轴线自转。
4. 地球可加入云层和月球运动；土星必须有星环；小行星带用粒子环表示。
5. 点击行星后，镜头平滑飞向该行星，选中轨道高亮，其它行星弱化。
6. 支持播放/暂停、1x/5x/20x 速度倍率、自由观察/自动巡航。
7. 鼠标悬停行星时，行星放大、发光，并出现 HUD 信息卡。

请输出：
1. 3D 科幻视觉定位
2. 网站信息架构
3. 页面清单
4. 每个页面的详细布局说明
5. 3D 场景结构
6. 行星运动规则
7. 镜头交互规则
8. 组件体系
9. 动画时序
10. 视觉风格规范
11. 移动端降级方案
12. 前端实现建议
13. 示例文案
14. 验收标准

前端建议技术栈：
Vue 3 或 React 18、TypeScript、Vite、Three.js、Tailwind CSS、Framer Motion/GSAP 可选、lucide 图标。

注意：
不要只写概念，要写清楚页面怎么布局、元素怎么运动、用户怎么交互、前端大概怎么实现。
```

## 二、Codex 开发提示词

```text
请基于太阳系科普网站产品方案，开发一个可运行的前端 Demo。

项目要求：
1. 创建独立项目目录 solar-system-demo，不要混入其它业务系统代码。
2. 使用 Vue 3 + TypeScript + Vite。
3. 使用 Three.js 实现首页和总览页的真 3D 太阳系。
4. 使用 CSS/Tailwind 实现深空科幻视觉、HUD 面板、玻璃拟态卡片和响应式布局。
5. 使用 lucide-vue-next 或等价图标库实现导航和按钮图标。

必须完成页面：
1. /solar：首页 3D Hero
2. /solar/overview：太阳系 3D 总览
3. /solar/planets：行星图鉴
4. /solar/planet/:id：行星详情页

3D 太阳系必须包含：
1. 太阳
2. 八大行星
3. 每颗行星的轨道
4. 小行星带
5. 星空粒子背景
6. 行星标签
7. HUD 数据浮层
8. 点击行星聚焦镜头
9. 拖拽旋转视角
10. 滚轮缩放

行星运动规则：
1. 每颗行星拥有 orbitRadius、orbitSpeed、rotationSpeed、size、tilt。
2. 每帧更新位置：
   x = cos(time * orbitSpeed) * orbitRadius
   z = sin(time * orbitSpeed) * orbitRadius
3. 每颗行星必须自转。
4. 内侧行星公转更快，外侧行星更慢，但采用科普展示比例，不追求真实比例。
5. 支持暂停/播放和 1x/5x/20x 速度倍率。
6. 暂停时公转停止，自转可保留轻微运动。

交互要求：
1. 鼠标悬停行星：行星高亮、放大、显示标签或 HUD。
2. 点击行星：镜头平滑靠近该行星，右侧展示 HUD 数据面板。
3. 双击或点击返回：回到太阳系全景。
4. 总览页提供控制器：播放/暂停、速度倍率、轨道显示、标签显示、自动巡航、比例模式。
5. 移动端降低粒子数量和复杂动画，保证不卡顿。

视觉要求：
1. 背景：深空黑 #020617。
2. 辅色：宇宙蓝 #0F172A、星云紫 #581C87、科技青 #22D3EE、太阳金 #FACC15。
3. 玻璃面板：rgba(15, 23, 42, 0.55)、backdrop-filter: blur(18px)、1px 半透明边框。
4. 太阳要足够亮，首页不能看不清。
5. 效果要高级克制，不要廉价霓虹灯。

数据要求：
1. 行星数据必须使用“约”表达不精确数据。
2. 不要胡编科普数据。
3. 每颗行星需要：名称、英文名、一句话特点、标签、平均距日距离、公转周期、自转周期、直径、温度范围、卫星数量、是否有光环、科普介绍。

工程要求：
1. README.md 写清项目介绍、功能亮点、技术栈、页面路径、本地运行、构建和目录结构。
2. .gitignore 排除 node_modules、dist、.env.local、日志和 zip。
3. package.json 只保留太阳系 Demo 必需依赖，不要包含无关业务依赖。
4. npm run build 必须通过。
5. 完成后检查仓库不要包含其它业务系统目录、接口、mock 或文档。

验收标准：
1. 打开 /solar 可以看到动态 3D 太阳系。
2. 太阳清晰发光，行星沿轨道运行。
3. 总览页可以拖拽、缩放、点击行星聚焦。
4. 行星图鉴和详情页可正常访问。
5. 移动端布局不重叠。
6. npm run build 通过。
7. GitHub 仓库只包含 solar-system-demo 相关内容。
```

## 三、交付检查清单

```text
1. 是否是独立项目目录 solar-system-demo？
2. 是否没有混入其它业务系统代码？
3. README.md 是否完整？
4. package.json 依赖是否精简？
5. .gitignore 是否排除 node_modules、dist、环境变量和日志？
6. /solar 是否能正常打开？
7. 太阳是否足够亮？
8. 行星是否有自转和公转？
9. 总览页是否支持拖拽、缩放、点击聚焦？
10. 图鉴页和详情页是否能跳转？
11. npm run build 是否通过？
12. GitHub 远端是否只包含太阳系 Demo？
```

## 四、给同事的使用建议

1. 先把 Product Design 提示词发给设计工具或设计同事，用于确认产品结构、视觉方向和交互要求。
2. 再把 Codex 开发提示词发给开发工具，用于落地可运行 Demo。
3. 开发完成后按“交付检查清单”逐项验收。
4. 如果继续迭代，可以单独追加需求，例如“加强移动端性能”“增加知识专题页”“增加行星测验”“接入真实 NASA 图片素材”。
