# AI 手势控制太阳系功能设计方案

## 1. 功能定位

`AI 手势控制` 是太阳系 3D 科普网站的沉浸式交互亮点。它通过摄像头识别用户手势，让用户在不使用鼠标的情况下控制动态 3D 太阳系，实现移动光标、选择行星、旋转视角、缩放镜头、切换行星、触发粒子爆炸和查看 HUD 数据。

体验目标不是“网页加摄像头”，而是让用户像操作科幻电影中的全息星图驾驶舱一样，用抬手、捏合、挥动、握拳等动作操控太阳系模型。

建议模块名称：

- AI 手势控制
- 星际手势操控
- 视觉手势探索模式
- Gesture Navigation

## 2. 用户使用流程

1. 用户进入 `/solar` 首页或 `/solar/overview` 总览页。
2. 点击蓝紫色发光按钮 `开启手势探索`。
3. 页面弹出 `CameraPermissionPanel`，说明摄像头用途、隐私策略和可关闭方式。
4. 用户授权摄像头。
5. 右上角出现摄像头预览窗口，叠加手部关键点骨架。
6. 页面出现蓝色发光虚拟光标，跟随食指移动。
7. `GestureHUD` 显示当前状态，例如 `已识别手部 / 当前手势：捏合选择 / 置信度 92%`。
8. 用户用手势操作 3D 太阳系。
9. 用户可随时关闭手势控制，回到鼠标模式。
10. 如果拒绝授权或设备性能不足，系统自动回退到鼠标控制，并可进入 `演示模式`。

## 3. 手势与操作映射表

| 手势 | 识别方式 | 页面效果 | 3D 效果 | 前端事件 | 优先级 |
|---|---|---|---|---|---|
| 食指移动 | 使用 indexTip 坐标映射屏幕坐标 | 显示蓝色虚拟光标，经过按钮/卡片/HUD 时高亮 | 光标射线命中行星时行星描边发光，轨道轻微亮起 | `gesture:pointerMove` | 低 |
| 食指 + 拇指捏合 | thumbTip 与 indexTip 距离小于阈值并 debounce | 等同点击按钮、卡片或行星 | 命中行星时镜头飞向行星，轨道增强，生成能量波 | `gesture:pinchSelect` | 高 |
| 张开手掌 | 五指伸展，掌心面积稳定 | 进入扫描模式，附近 HUD 显示简短信息 | 行星表面扫过扫描线，轨道出现全息脉冲 | `gesture:scanStart` | 中 |
| 张开手掌停留 1 秒 | openPalm 持续超过 1000ms | 展开深度信息卡 | 行星外圈生成慢速扫描环 | `gesture:deepScan` | 中 |
| 握拳 | 指尖靠近掌心，持续多帧 | 触发粒子爆炸或返回全景 | 光标处球形粒子爆炸；靠近行星时生成护盾波纹 | `gesture:fistBurst` | 高 |
| 左挥手 | 手掌中心短时间快速向左位移 | 切换上一颗行星，HUD 更新 | 镜头沿轨道飞向上一颗行星，产生横向光速线 | `gesture:swipePrev` | 高 |
| 右挥手 | 手掌中心短时间快速向右位移 | 切换下一颗行星，HUD 更新 | 镜头沿轨道飞向下一颗行星，星尘拖尾 | `gesture:swipeNext` | 高 |
| 双指张开 | 食指与中指距离增大 | 镜头拉近 | 相机向目标点推进，星空纵深增强 | `gesture:zoomIn` | 中 |
| 双指靠近 | 食指与中指距离减小 | 镜头拉远 | 相机远离目标点，轨道全景变清晰 | `gesture:zoomOut` | 中 |
| 张开手掌左右移动 | openPalm 状态下掌心横向移动 | 控制太阳系视角旋转 | 太阳系容器绕 Y 轴平滑旋转，带惯性 | `gesture:orbitRotate` | 中 |

## 4. 页面级使用方式

### 4.1 首页 Hero

入口：

- Hero 主按钮组新增 `开启手势探索`。
- 按钮使用摄像头/手势图标，蓝紫色发光胶囊样式。

手势行为：

- 食指移动：控制虚拟光标，扫过太阳、行星、按钮时产生高亮。
- 捏合行星：进入该行星详情页 `/solar/planet/:id`。
- 张开手掌：太阳、轨道线、行星标签进入扫描模式。
- 握拳：在光标位置触发星尘爆炸，作为趣味反馈。
- 左右挥手：在首页高亮不同推荐行星，Hero 文案旁边的行星芯片同步切换。

3D 效果：

- 捏合命中行星时，目标行星外圈生成环形粒子，镜头短暂推进后执行路由跳转。
- 张开手掌扫描太阳时，太阳日冕亮度提升 15%-25%，轨道出现一圈 HUD 扫描线。
- 握拳爆炸持续 0.8-1.2 秒，不遮挡标题主体。

### 4.2 太阳系总览页

入口：

- 总览页控制栏新增 `手势模式` 开关。
- 开启后保留鼠标控制作为备用，手势控制成为主要输入。

手势行为：

- 食指移动：虚拟光标在 3D 星图上移动，射线检测行星。
- 捏合：选择行星，打开右侧 HUD 数据面板。
- 左右挥手：上一颗/下一颗行星。
- 双指张合：缩放相机。
- 张开手掌左右移动：旋转整个太阳系视角。
- 握拳：如果已聚焦行星，则返回全景；如果未聚焦，则触发粒子爆炸。

3D 效果：

- 被选择行星放大 1.12 倍并增强 emissive。
- 目标轨道线亮度提升，其它轨道透明度降低。
- 镜头使用 900-1400ms ease-out 飞行到行星侧前方。
- 右侧 HUD 面板同步显示行星名称、距日距离、公转周期、自转周期、直径、温度、卫星数量和是否有光环。

### 4.3 行星详情页

入口：

- 行星详情页顶部 3D 场景旁显示小型 `手势控制` 按钮。

手势行为：

- 张开手掌左右移动：旋转当前行星视角。
- 双指张合：靠近或远离行星表面。
- 捏合数据卡：展开该数据项的科普解释。
- 握拳：触发行星周围能量环。
- 左右挥手：切换上一颗/下一颗行星详情。

行星专属反馈：

- 地球：捏合时云层短暂高亮，月球轨道出现蓝白粒子环。
- 火星：握拳时出现红色沙尘扩散和探测任务标记点。
- 土星：双指缩放时镜头沿星环侧面掠过，星环颗粒产生流动。
- 木星：扫描时大红斑区域出现 HUD 锁定框。

## 5. 摄像头权限与隐私设计

权限面板必须明确展示：

- 摄像头仅用于本地浏览器手势识别。
- 摄像头画面不会上传服务器。
- 不保存、不截图、不录制摄像头画面。
- 用户可随时关闭手势控制。
- 拒绝授权后自动回到鼠标控制模式。
- 展厅或汇报场景可使用 `演示模式`，不需要摄像头。

状态处理：

| 状态 | UI 文案 | 系统行为 |
|---|---|---|
| 未开启 | `鼠标控制模式` | 不初始化摄像头和模型 |
| 等待授权 | `需要摄像头权限以识别手势` | 展示权限说明 |
| 已开启检测中 | `正在检测手部` | 显示预览窗口和骨架层 |
| 已识别手部 | `已识别手部` | 启用虚拟光标和手势事件 |
| 未检测到手 | `请将手掌放入摄像头画面中` | 不触发关键操作 |
| 置信度不足 | `手势置信度较低` | 只移动光标，不执行选择/切换 |
| 暂停 | `手势控制已暂停` | 停止识别循环，保留摄像头可选 |
| 关闭 | `手势控制已关闭` | 停止摄像头轨道，释放媒体流 |
| 摄像头异常 | `摄像头不可用，请使用鼠标模式` | 回退鼠标控制 |

## 6. 手势 HUD 界面设计

`GestureHUD` 固定在右上角，覆盖在 3D 场景上方。

内容：

- 摄像头状态：开启 / 关闭 / 异常。
- 当前手势：例如 `捏合选择`、`扫描模式`、`握拳爆炸`。
- 置信度：0%-100%，低于阈值时变为黄色。
- 当前目标：悬停或选中的行星名称。
- 模式切换：鼠标模式 / 真实手势模式 / 演示模式。

视觉：

- 半透明玻璃拟态面板。
- 顶部细线显示识别信号强度。
- 手势成功触发时，HUD 外框产生一次蓝色脉冲。
- 摄像头异常时使用红色警示点，但不要大面积红色污染科幻视觉。

## 7. 粒子效果设计

### 7.1 捏合点击粒子波

触发：`gesture:pinchSelect`

效果：

- 光标位置生成蓝色能量环。
- 能量环从 0.2 倍扩展到 1.8 倍并淡出。
- 周围散出少量星尘粒子。
- 如果命中行星，行星表面出现扫描光圈。

建议参数：

- 持续时间：0.45-0.7 秒
- 粒子数：低配 24，高配 80
- 颜色：`#22D3EE`、`#818CF8`

### 7.2 握拳爆炸粒子

触发：`gesture:fistBurst`

效果：

- 光标世界坐标处出现球形粒子爆炸。
- 粒子高速向外扩散后减速淡出。
- 爆炸后留下一层短暂星尘雾。
- 靠近行星时，额外生成行星护盾波纹。

建议参数：

- 持续时间：0.8-1.5 秒
- 粒子数：低配 80，高配 260
- 颜色：科技青、星云紫、太阳金
- 最大半径：当前相机距离的 3%-8%

### 7.3 行星聚焦粒子环

触发：捏合选择行星。

效果：

- 目标轨道线突然亮起。
- 行星外圈生成环形粒子。
- 镜头飞行期间粒子沿轨道形成拖尾。
- 镜头到达后粒子收束到 HUD 面板边缘。

### 7.4 星系切换粒子流

触发：`gesture:swipePrev` / `gesture:swipeNext`

效果：

- 画面出现横向光速线。
- 当前行星到目标行星之间出现一条发光轨迹。
- HUD 数据在飞行结束后更新，避免中途闪烁。

### 7.5 手势光标星尘拖尾

触发：食指移动。

效果：

- 虚拟光标是 2D DOM 层的蓝色光点。
- 拖尾由最近 8-16 个指尖坐标生成。
- 移动速度越快，拖尾越长、透明度越低。
- 命中 3D 行星时，行星上额外产生微弱的 3D 粒子吸附。

## 8. 与 Three.js 3D 场景的结合方式

手势输入分为 2D 屏幕层和 3D 世界层：

1. MediaPipe 输出手部关键点，得到归一化坐标。
2. `GestureMapper` 将 indexTip 映射为屏幕坐标。
3. `GestureCursor` 用屏幕坐标显示 2D 发光光标。
4. `GestureParticleBridge` 将屏幕坐标转换为 Three.js NDC 坐标。
5. 通过 `Raycaster` 检测是否命中行星 Mesh。
6. 如果命中行星，粒子效果附着到该行星附近或轨道上。
7. 如果没有命中行星，在相机前方固定深度的平面上计算世界坐标，生成空间粒子。
8. 粒子系统记录生命周期，播放结束后自动从 Scene 中移除。

屏幕坐标到世界坐标建议：

```ts
const ndc = new THREE.Vector2(
  (screenX / width) * 2 - 1,
  -(screenY / height) * 2 + 1
)

raycaster.setFromCamera(ndc, camera)

const hit = raycaster.intersectObjects(planetMeshes, true)[0]

if (hit) {
  spawnParticle({ type: 'planet-focus', position: hit.point, targetPlanetId })
} else {
  const fallbackPoint = raycaster.ray.origin
    .clone()
    .add(raycaster.ray.direction.clone().multiplyScalar(defaultDepth))
  spawnParticle({ type: 'space-burst', position: fallbackPoint })
}
```

联动规则：

- 行星选中：轨道高亮、行星材质 emissive 增强、HUD 更新、聚焦粒子环播放。
- 扫描模式：轨道线变亮，行星标签展开，HUD 面板出现扫描线。
- 握拳爆炸：如果距离行星小于命中阈值，播放行星护盾；否则播放空间爆炸。
- 左右挥手：相机飞行、星际拖尾、目标 HUD 延迟更新。

## 9. 前端技术实现建议

推荐技术路线：

- React 18 + TypeScript + Vite
- React Three Fiber / Three.js
- Drei 用于相机、标签、辅助控制
- MediaPipe Hand Landmarker 优先，TensorFlow.js Handpose 作为备选
- Framer Motion 或 GSAP 用于 HUD 和页面 UI 动效
- 粒子系统使用 Three.js `BufferGeometry + PointsMaterial` 或 GPU 粒子方案

当前 Vue Demo 若继续迭代，也可以沿用 Vue 3 + Three.js，手势层作为独立组合式模块实现。核心设计不依赖具体框架。

## 10. 前端组件拆分

| 组件 | 类型 | 职责 | 输出/事件 |
|---|---|---|---|
| `GestureControlProvider` | 状态容器 | 管理摄像头权限、模型初始化、识别循环、状态输出 | `gestureState`、`landmarks`、`confidence` |
| `CameraPermissionPanel` | UI | 展示权限说明，引导开启/关闭摄像头 | `onAllow`、`onDeny` |
| `CameraPreviewHUD` | UI | 显示小型摄像头预览和手部骨架线 | 无 |
| `GestureCursor` | UI | 显示虚拟光标和星尘拖尾 | `pointerMove` |
| `GestureHUD` | UI | 显示当前手势、置信度、目标行星、模式 | `onModeChange` |
| `GestureTutorialPanel` | UI | 首次教学说明，可缩小为提示条 | `onDismiss` |
| `GestureMapper` | Logic | 将关键点转换为点击、扫描、挥手、缩放等动作 | `gesture:*` |
| `GestureParticleBridge` | 3D Bridge | 将手势事件转成 Three.js 世界坐标粒子事件 | `spawnParticle` |
| `ParticleExplosion` | 3D | 播放爆炸、能量环、星尘、轨道拖尾 | 生命周期完成后销毁 |
| `GestureModeSwitch` | UI | 鼠标模式、真实手势模式、演示模式切换 | `modeChange` |
| `GestureDebugPanel` | Dev | 显示关键点、坐标、事件、阈值，生产隐藏 | 无 |

建议事件接口：

```ts
type GestureEvent =
  | { type: 'pointerMove'; screen: { x: number; y: number }; confidence: number }
  | { type: 'pinchSelect'; screen: { x: number; y: number }; targetId?: PlanetId }
  | { type: 'scanStart'; screen: { x: number; y: number } }
  | { type: 'deepScan'; targetId?: PlanetId }
  | { type: 'fistBurst'; screen: { x: number; y: number }; targetId?: PlanetId }
  | { type: 'swipePrev' }
  | { type: 'swipeNext' }
  | { type: 'zoom'; delta: number }
  | { type: 'orbitRotate'; deltaX: number }
```

## 11. 手势识别逻辑

### 11.1 捏合判断

- 计算 `thumbTip` 与 `indexTip` 的距离。
- 距离小于阈值进入 pinch。
- 从非 pinch 到 pinch 的瞬间触发一次点击。
- 使用 350-600ms 冷却，避免连续误触。
- 置信度低于 0.75 时不触发。

### 11.2 张开手掌判断

- 五个手指伸展。
- 指尖到掌心距离大于阈值。
- 手掌面积较大且连续稳定 8-12 帧。
- 持续超过 1 秒触发 `deepScan`。

### 11.3 握拳判断

- 多个指尖靠近掌心。
- 手指弯曲比例达到阈值。
- 连续 6-10 帧识别为 fist 才触发。
- 加 1.2 秒冷却，避免爆炸效果过密。

### 11.4 左右挥手判断

- 使用掌心中心点横向位移。
- 短时间内位移超过阈值并速度足够快。
- 判断方向为 left/right。
- 加 900-1300ms 冷却，避免重复切换。

### 11.5 双指缩放判断

- 检测 indexTip 与 middleTip 距离变化。
- 距离增大触发 zoomIn，距离减小触发 zoomOut。
- 使用 EMA 平滑，避免相机抖动。
- 只有双指稳定伸展时启用。

### 11.6 虚拟光标

- 使用 indexTip 坐标。
- 摄像头横向镜像，符合用户直觉。
- 使用 EMA 或 lerp 平滑。
- 坐标映射为屏幕坐标。
- 如果未检测到手，光标淡出，不触发操作。

## 12. 手势冲突处理规则

优先级从高到低：

1. 摄像头权限、模式开关、关闭控制。
2. 捏合选择。
3. 握拳爆炸 / 返回全景。
4. 左右挥手切换。
5. 双指缩放。
6. 张开手掌扫描。
7. 食指移动光标。

规则：

- 置信度不足时只允许移动光标，不触发关键操作。
- 捏合优先于悬停扫描。
- 握拳优先于张开手掌扫描。
- 挥手必须满足速度阈值和冷却时间。
- 双指缩放只在双指稳定时启用。
- 没有明确手势时，系统只更新虚拟光标和 hover 状态。

## 13. 演示模式设计

`演示模式` 用于没有摄像头、不方便授权、展厅大屏或录屏汇报场景。

特点：

- 不需要摄像头权限。
- 使用预设手势轨迹驱动虚拟光标。
- 自动演示：移动光标、捏合选择地球、挥手切换火星、握拳爆炸、双指缩放。
- HUD 显示 `演示模式：模拟手势数据`。
- 支持用户随时切换到真实手势模式。

演示脚本建议：

1. 0-3 秒：光标从标题区滑向太阳系。
2. 3-5 秒：张开手掌扫描太阳。
3. 5-7 秒：捏合选择地球，镜头飞向地球。
4. 7-10 秒：右挥手切换火星。
5. 10-12 秒：握拳触发粒子爆炸。
6. 12-15 秒：返回全景。

## 14. 性能优化与降级

识别与渲染：

- 手势识别限制在 15-30 FPS。
- 3D 渲染优先保持 60 FPS 或设备可承受的稳定帧率。
- 手势关键点计算与 3D 渲染解耦。
- 只在手势模式开启时加载 MediaPipe 模型。
- 非当前页面暂停识别。
- 页面隐藏时停止摄像头媒体流和动画。

粒子：

- 粒子数量按设备性能分级。
- 高性能模式：爆炸 200-300 粒子，轨道拖尾和 Bloom 开启。
- 标准模式：爆炸 80-160 粒子，限制 Bloom 和景深。
- 低性能模式：爆炸 24-80 粒子，关闭复杂后期和拖尾。
- 粒子生命周期结束后必须销毁 BufferGeometry 和 Material。

移动端：

- 默认关闭真实摄像头手势模式。
- 提供演示模式和鼠标/触摸控制。
- 降低摄像头预览分辨率。
- 关闭 Bloom、景深、复杂粒子爆炸。
- 保留核心交互：触摸选择行星、双指缩放、拖拽旋转。

## 15. 交付给前端开发的实现说明

开发顺序建议：

1. 实现 `GestureControlProvider`：打开摄像头、加载模型、输出手部关键点。
2. 实现 `GestureCursor`：食指控制虚拟光标，支持拖尾。
3. 实现 `GestureMapper`：先完成 pointerMove、pinchSelect、fistBurst 三个核心手势。
4. 接入 `SolarSystemScene`：用 Raycaster 判断光标是否命中行星。
5. 实现 `ParticleExplosion` 和 `GestureParticleBridge`：把捏合和握拳映射到 3D 粒子。
6. 增加 `GestureHUD` 与 `CameraPreviewHUD`。
7. 补齐挥手切换、双指缩放、张开手掌扫描。
8. 增加演示模式。
9. 做性能分级和移动端降级。
10. 完成隐私说明和异常处理。

最小可验收版本：

- 可以开启/关闭手势控制。
- 摄像头预览和手部骨架正常显示。
- 食指可控制虚拟光标。
- 捏合可选择行星，并触发镜头聚焦。
- 握拳可在 3D 场景中触发粒子爆炸。
- HUD 能显示当前手势、置信度和摄像头状态。
- 拒绝摄像头权限后能回到鼠标控制。
- 粒子效果能够自动销毁，不造成持续内存增长。

## 16. 验收标准

硬性验收：

- 视觉手势可以替代鼠标完成核心太阳系探索。
- 食指移动可以控制虚拟光标。
- 捏合可以触发点击/选择。
- 捏合行星后镜头可以飞向目标行星。
- 握拳可以触发 3D 粒子爆炸。
- 粒子效果可以通过 Three.js 世界坐标生成，而不只是 2D 页面动画。
- 摄像头权限与隐私说明清晰可见。
- 拒绝授权时不影响鼠标模式使用。
- 移动端和低性能设备有降级策略。

体验验收：

- 手势反馈要明显但克制，不遮挡科普内容。
- HUD 状态能帮助用户理解当前识别结果。
- 手势误触率可控，关键操作有 debounce 和冷却。
- 演示模式能完整展示手势控制亮点。
